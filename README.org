#+TITLE: Lispbar - A Pure Elisp Status Bar for X11 and Wayland
#+AUTHOR: Lispbar contributors

* Overview

Lispbar is a pure Emacs Lisp status bar that runs on every modern
graphical Emacs build.  It abstracts the display server behind a
pluggable backend protocol, so the same configuration works on:

- *EXWM* (X11) - workspaces, struts, dock window-type hints
- *Sway* (Wayland / wlroots) - monitors and workspaces via =swaymsg=
- *Hyprland* (Wayland) - monitors and workspaces via =hyprctl=
- *Plain Wayland* (PGTK Emacs, other compositors) - via
  =display-monitor-attributes-list=
- *Plain graphical Emacs* on X / NS / GTK as a last-resort fallback

Every Lisp symbol obeys GNU Elisp conventions
(=lispbar-= public, =lispbar--= private, lexical binding everywhere).

* Features

- Pure Elisp, no compiled C extensions
- Pluggable backend protocol (=cl-defgeneric= on EIEIO classes) -
  users can register new backends without modifying Lispbar
- Multi-monitor with per-monitor configuration
- EIEIO module system with priorities, dependencies, caching,
  inter-module messaging, and lifecycle hooks
- Beautiful rendering engine with overflow handling, dynamic width
  and theme-aware faces

* Requirements

- GNU Emacs 27.1 or later (29.1+ recommended for the PGTK build on
  Wayland)
- Optional: =exwm=, =swaymsg=, =hyprctl=

EXWM is *not* a hard dependency; it is loaded automatically only
when installed.

* Installation

#+begin_src elisp
(add-to-list 'load-path "/path/to/lispbar")
(require 'lispbar)
(lispbar-mode 1)
#+end_src

** Using straight.el

#+begin_src elisp
(straight-use-package
 '(lispbar :type git :host github :repo "theesfeld/lispbar"))
(lispbar-mode 1)
#+end_src

* Wayland Quick Start

Emacs cannot become a layer-shell surface on its own, so on Wayland
Lispbar relies on the compositor to pin its frame.  Two helpers
print ready-to-paste rules:

#+begin_src
M-x lispbar-wayland-print-sway-config       ; Sway / wlroots
M-x lispbar-wayland-print-hyprland-config   ; Hyprland
#+end_src

Copy the snippet into =~/.config/sway/config= or
=~/.config/hypr/hyprland.conf= and reload the compositor.  Then run
=M-x lispbar-mode=.

* Selecting a Backend

By default Lispbar autoselects the highest-priority backend whose
=lispbar-backend-available-p= returns non-nil.  Override via:

#+begin_src elisp
(setq lispbar-backend-choice 'sway)        ; force Sway backend
(setq lispbar-backend-choice 'auto)        ; restore autoselection
#+end_src

** Writing a Custom Backend

#+begin_src elisp
(defclass my-river-backend (lispbar-backend) ()
  "Backend for the river Wayland compositor.")

(cl-defmethod lispbar-backend-available-p ((_ my-river-backend))
  (and (getenv "WAYLAND_DISPLAY")
       (executable-find "riverctl")))

(cl-defmethod lispbar-backend-detect-monitors ((_ my-river-backend))
  ...)

(lispbar-backend-register
 (make-instance 'my-river-backend
                :name 'river :priority 75
                :features '(monitor-detect workspaces)))
#+end_src

That is the entire integration.  Lispbar will now offer =river= in
=lispbar-backend-choice= and call your methods through
=cl-defgeneric= dispatch.

* Configuration

#+begin_src elisp
(use-package lispbar
  :custom
  (lispbar-position 'top)
  (lispbar-height 28)
  (lispbar-default-modules '(workspace time window-title battery))
  :config
  (lispbar-mode 1))
#+end_src

* Creating Custom Modules

Modules are EIEIO objects.  The shortcut:

#+begin_src elisp
(lispbar-create-module
 'cpu-load
 (lambda ()
   (with-temp-buffer
     (insert-file-contents "/proc/loadavg")
     (format "Load %s" (car (split-string (buffer-string))))))
 :position 'right :priority 60 :update-interval 5.0)
#+end_src

* Architecture

| File                          | Purpose                                       |
|-------------------------------|-----------------------------------------------|
| =lispbar-backend.el=          | Generic protocol (cl-defgeneric + EIEIO)      |
| =lispbar-backend-wayland.el=  | Sway, Hyprland, generic Wayland backends      |
| =lispbar-backend-exwm.el=     | X11/EXWM backend (loaded when EXWM present)   |
| =lispbar-core.el=             | Frames, monitors, struts, hot-plug            |
| =lispbar-render.el=           | Layout engine                                 |
| =lispbar-modules.el=          | Module system                                 |
| =modules/*.el=                | Built-in modules (clock, battery, ...)        |
| =lispbar.el=                  | Top-level minor mode and user-facing API      |
